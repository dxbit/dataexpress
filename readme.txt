DataExpress - это простая программа для хранения и поиска информации.
Это может быть какая-нибудь коллекция, справочная информация, каталог, документы,
учетные данные и т. п. Программа имеет простой классический интерфейс и не требует
много времени на освоение. Данные вводятся в формы, которые вы создаете сами по
своему усмотрению. Для этого в программе предусмотрен дизайнер форм. Размещайте
компоненты на форме как вам удобно. Каждый компонент может хранить данные
определенного типа. На форме можно размещать следующие компоненты: надпись, текст,
число, дата, многострочный текст, флажок, список, список объектов, таблица, группа,
закладки. В программе предусмотренно несколько режимов просмотра: в виде таблицы,
в виде формы, в таблице и форме одновременно. С помощью фильтра можно отбирать
данные по определенным условиям. Есть и обычный поиск по определенным полям или
всем полям одновременно. Вы можете распечатывать информацию, используя шаблоны.
Шаблоны создаются в текстовом редакторе, способном сохранять в форматах DOCX, ODT,
WORD XML или HTML.
Функции программы:
	- хранение данных в базе данных;
	- ввод и редактирование данных в форме и таблице;
	- поиск данных по отдельным полям или всем полям одновременно;
	- фильтрация данных;
	- сортировка данных;
	- печать данных с использованием шаблонов;
	- импорт/экспорт данных в формат CSV;
	- просмотр данных в нескольких режимах;
	- запоминает значения фильтров, сортировку, ширину и позицию столбцов таблиц.
Если вы храните свои данные в блокноте или в Excel, или другой подобной программе,
вы можете рассмотреть DataExpress как более подходящую альтернативу для ваших задач.
Удачи!

Столкнулся с утечкой памяти при просмотре изображений в гриде. Память выделяется

связано с тем, что датасет хранит данные в своих буферах внутри.
У IBConnection есть метод LoadField, в который в качестве выходного
параметра передается CreateBlob. Если после выполнения этого метода CreateBlob=
True, то датасет выделяет для него память. И при вызове CreateBlobStream память
для блоб не выделяется, а берется указатель из буфера датасета. Но т. к. в
исходнике LoadField CreateBlob всегда False, то датасет не выделял для блоб
память, и при вызове CreateBlobStream память для блоб каждый раз выделялась
снова.

Обнаружил неприятную особенность Pascal Script, связанную с делением чисел.
Деление целых чисел в результате дает всегда целое число, дробная часть
отбрасывается. Причем не зависит какая операция используется: / или div.
Это может оказаться полной неожиданностью для разработчика, который привык к
Delphi или Lazarus, где подобная операция в результате дает дробное число.
Поэтому при делении нужно всегда указывать дробную часть даже если ее нет.
Причем операции / и div ведут себя абсолютно одинаково. Т. е. казалось бы
целочисленное деление в результате может дать дробное число.
