Каждый компонент имеет определенное назначение. Все компоненты можно разделить на несколько групп:

Поля - эти компоненты предназначены для хранения данных определенного типа (число, текст, дата и т. д.). Они задают структуру базы данных. При добавлении такого компонента в таблице базы данных (с которой ассоциирована форма) создается новое поле (или группа полей). При удалении компонента поле удаляется и данные, хранящиеся в этом поле, тоже. Помните об этом, когда удаляете компоненты, т. к. можно случайно удалить нужные данные. (Создание/удаление полей в базе данных осуществляется не в момент добавления/удаления компонента, а при сохранении изменений в дизайнере.)

Контейнеры - эти компонеты не ассоциированы с полями базы данных, а предназначены для группировки компонент по определенному смыслу. К таким компонентам относятся группы, закладки и сама форма. Шрифт контейнера влияет на шрифт компонентов в контейнере. При удалении контейнера все компоненты, находящиеся в нем тоже удаляются.

Оформление - эти компоненты используются для визуального оформления формы для повышения ее читаемости. К таким компонентам относятся надписи, фоновые изображения и фигуры.

Подчиненные данные - эта группа представлена одним компонентом Таблица. Она создает отдельную подчиненную форму, данные которой зависят от родительской формы.

Обработка данных - в этой группе только один компонент Запрос. Это очень важный компонент, который позволяет отображать и обрабатывать данные других форм.
