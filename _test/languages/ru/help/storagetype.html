Первый способ подходит для небольших файлов или если нет возможности использовать общую папку для хранения при сетевом доступе.

Второй способ подходит для больших файлов. Файл копируется в указанную папку. Если файл с таким именем существует, то к имени добавляется число, делая его имя уникальным. Может не подойти при сетевом доступе, если нет возможности создать папку на сетевом диске. Папка может иметь относительный путь, т. е. находится в папке программы.

Третий способ просто прописывает ссылку на файл в базе данных. Такой способ годится для локальных баз и когда файлы находятся на месте постоянно. Третий способ не подходит для DataExpress Server.

В случае изображений, миниатюра хранится всегда в базе.
